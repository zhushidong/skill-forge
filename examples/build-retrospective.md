## 第一章：这玩意儿能帮我什么？

### 场景一：把老同事的"绝招"学过来

**你的日常**：团队里有个销冠，每次客户说"太贵了"，他总有办法搞定。你问他怎么说的，他说"就那样说啊"。你观察了十次，还是学不会。

**用 Skill Forge 怎么做**：

```
1. 把销冠的一次成功对话录下来，写成聊天记录（就像写微信对话）
2. 丢给 Skill Forge → "帮我提炼成 Skill"
3. 得到一个"价格异议应对 Skill"
4. 打开它，里面清楚写着：
   - 什么情况下用（客户说太贵了、客户拿竞品压价）
   - 客户会说什么（"预算不够"、"别人家便宜"）
   - 你应该怎么回（先认同→再了解预算→展示价值→给方案）
   - 绝对不能做什么（直接降价、贬低竞品）
5. 照着练几遍 → 变成你自己的本事
```

**这就叫"把经验结构化"——从"我学不会他怎么说的"到"我照着步骤走就行"。**

### 场景二：把一份超专业培训材料变成随时能用的话术

**你的日常**：公司花大价钱请了咨询公司，出了一份 50 页的培训材料。你看了三页就困了。材料写得再好，记不住也用不上。

**用 Skill Forge 怎么做**：

```
1. 把培训材料的重点部分复制出来，存成一个 .md 文件
2. 丢给 Skill Forge → "帮我炼成 Skill"
3. 得到一份结构清晰的 Skill，只有一页纸的长度
4. 核心内容都在里面：适用场景、客户信号、话术示例
5. 演练几遍，效果比看 50 页材料好得多
```

**这就叫"知识压缩"——把 50 页的材料压缩成一张"武功秘籍"。**

### 场景三：把一周的聊天记录变成自己的话术库

**你的日常**：这周跟 20 个客户聊了天。有成的有没成的。你感觉学到了很多，但说不清到底学到了什么。

**用 Skill Forge 怎么做**：

```
1. 挑一个"感觉聊得不错"的聊天记录
2. 丢给 Skill Forge 复盘 → 自动分析：
   - 你哪些话说得好（下次继续用）
   - 哪些话说得不好（下次避免）
   - 这个客户属于什么类型
3. 系统自动更新你的 Skill，把新经验加进去
4. 版本从 v1.0 变成 v1.1
5. 你的话术库在不断进化
```

**这就叫"经验沉淀"——每次跟客户聊完都在进步，而不是聊完就忘。**

### 场景四：把 AI 智能体的设定"吸收"过来

**你的日常**：你在 Workbuddy 上看到一个很牛的 AI 智能体设定文件，或者同事分享了一个 prompt。你想把它变成自己能用的东西。

**用 Skill Forge 怎么做**：

```
1. 把那个设定文件（JSON/YAML/MD 格式都行）传给 Skill Forge
2. 它会"熔炼"这个外部资产 → 分析里面包含了什么能力
3. 输出一份结构化的 Skill，吸收了外部资产的核心策略
4. 你可以接着演练、修改、变成自己的
```

**这就叫"知识迁移"——把别人的"底牌"学过来。**

## 第二章：三分钟快速上手

### 第一步：装 Python（你只需要做一次）

**Python 是什么？** 你可以把它理解成一个"翻译器"——Skill Forge 是用 Python 写的，所以你得先装这个翻译器才能运行它。

**安装步骤（包教包会）**：

```
1. 打开浏览器，地址栏输入 https://www.python.org/downloads/
2. 会看到一个黄色大按钮写着 "Download Python 3.12.x"
3. 点击它，下载一个安装包
4. 双击运行安装包
5. ⚠️ 重要：安装界面最下面有一个 "Add Python to PATH"
   一定要打勾！不打勾后面会报错
6. 然后点 "Install Now"，等一两分钟
7. 安装完成，关掉窗口
```

**验证装没装成功**：
```
1. 按 Win+R（Windows 键 + R 键）
2. 输入 cmd，回车
3. 在黑色窗口里输入：python --version
4. 如果看到 "Python 3.12.x" 之类的，说明装成功了
```

### 第二步：装 Skill Forge

```
1. 打开浏览器，进入 https://github.com/zhushidong/skill-forge
2. 点绿色的 "Code" 按钮 → 点 "Download ZIP"
3. 解压到你喜欢的地方，比如 D:\skill-forge
4. 打开 cmd（Win+R → cmd → 回车）
5. 输入：cd /d D:\skill-forge （看你解压到哪）
6. 输入：pip install -e .
   这行命令的意思是"安装当前目录下的这个工具"
7. 等它跑完，出现 Successfully installed 就装好了
```

### 第三步：初始化

```
1. 在 cmd 里输入：skill-forge init
2. 看到"初始化完成"就对了
3. 这会创建一些文件夹，用来放你的数据
```

**装好了！你现在可以开始用这个工具了。**

## 第三章：配合 Workbuddy（豆包）使用

### 核心思路

Skill Forge 的很多功能需要 AI 帮忙（比如"把案例提炼成 Skill"）。它可以用两种方式工作：

| 方式 | 适合谁 | 怎么做 |
|------|--------|--------|
| **自动模式** | 有 OpenAI 账号 | AI 自动完成，你等着就行 |
| **手动模式** | 用 Workbuddy/通义/Kimi | 复制提示词 → 粘贴到 AI 工具 → 把结果存回来 |

**你用 Workbuddy，就用手动模式。** 下面我详细说。

### 手动模式操作流程

以"提炼 Skill"为例：

```
# 第一步：在 cmd 里输入（不讲怎么输入命令行的话你可以直接打开cmd复制）
skill-forge distill --material 材料ID --problem "要解决的问题"

# 第二步：工具会输出一大段"提示词"
# 你会看到：
## 未检测到 OPENAI_API_KEY
## 你可以将以下 Prompt 复制到任意大模型中使用：
### ...很长的一段内容...

# 第三步：全选复制这段内容
# 第四步：打开 Workbuddy（豆包）
# 第五步：粘贴进去，发送
# 第六步：Workbuddy 会给你一段回答，包含 YAML 格式的 Skill
# 第七步：把这段回答复制回来，存成一个文件放到 data/skills/draft/ 目录下
```

**所有需要 AI 的命令（distill、melt、drill、review、recommend）都是这个流程。** 工具把"炼丹配方"给你，你自己找 AI 去炼，炼好了再拿回来。

## 第四章：走一遍完整流程

我带你把整个流程走一遍。**不需要懂代码，跟着做就行。**

### 准备：一个案例文件

Skill Forge 自带了一个示例文件，在 `samples/case-price.md`。打开看看：

```
# 客户嫌贵案例

## 背景
客户：张总，中小企业老板
产品：999元线上课程

## 对话记录
销售：张总您好，看了我们的课程介绍，感觉怎么样？
客户：内容还行，就是价格有点贵。
... 后面还有很长 ...
```

这是一次失败的销售——客户说贵，销售直接降价，最后客户说"再考虑"，然后流失了。

现在我们要把这个失败案例"炼"成一个 Skill——下次再遇到这种情况，就知道该怎么说了。

### 第一步：导入案例

```bash
skill-forge ingest --type case --title "客户嫌贵案例" --file samples/case-price.md
```

输出：
```
已导入资料：
  - ID: material-20260617-153000
  - 路径: data/materials/cases/material-20260617-153000.md
```

**这个命令做了什么？**
- 读了你提供的文件
- 存到 `data/materials/cases/` 目录下
- 给了它一个编号（ID），方便后面引用

### 第二步：提炼成 Skill

```bash
skill-forge distill --material material-20260617-153000 --problem "客户嫌贵怎么应对"
```

如果你没设 API Key，会输出一段很长的提示词。**复制它**，粘贴到 Workbuddy 里，让 AI 帮你生成 Skill。

AI 会给你一段内容，格式大概是：

```yaml
---
id: skill-xxx
name: 客户嫌贵应对 Skill
status: draft
applicable_scenarios:
  - 客户直接说"太贵了"
  - 客户拿竞品低价对比
customer_signals:
  - "太贵了"
  - "预算不够"
strategy:
  diagnosis: 判断客户是预算不足还是价值认知不足
  response_quality: 先用问题确认真实顾虑
  next_step_control: 每次回应都争取一个下一步
  risk_control: 不要直接降价
forbidden_behaviors:
  - 未经审批直接降价
  - 贬低竞品
---
```

把这段内容存成一个 `.md` 文件，放到 `data/skills/draft/` 目录下。

**现在你有了第一个 Skill！** 它是一个结构化的业务应对策略。

### 第三步：演练（自测）

```bash
skill-forge drill --skill skill-xxx-001 --persona "预算不足型客户" --rounds 3
```

这个命令会：
- 扮演一个"预算不足型客户"
- 跟你对话 3 轮
- 最后给你打分（0-100分）

**如果没 API Key**：同样会输出提示词，复制到 Workbuddy，让 AI 扮演客户跟你对话。

打完分之后，系统会自动更新你的 Skill 指标——演练次数 +1，平均分更新。

### 第四步：复盘（真实对话）

```bash
skill-forge review --file samples/current-chat.md --result "推进" --skill skill-xxx-001
```

复盘会分析你的真实聊天记录，输出：
- 你执行了几步 Skill（有没有跑偏）
- 结果是怎样的（推进了还是失败了）
- 有没有什么可以改进的

复盘后系统会自动更新 Skill 的实战次数和战绩。

### 第五步：看 Skill 状态

```bash
skill-forge search
```

会列出你所有的 Skill：

```
找到 1 个匹配 Skill

  1. [draft] 客户嫌贵应对 Skill (匹配分: 0.5)
     ID: skill-xxx-001    版本: 1.0.0
     战绩: 暂无实战
```

随着你不断演练和复盘，这个 Skill 会自动"晋升"：
- 演练 3 次以上 + 平均分 ≥ 60 → **trained**（已训练）
- 实战 1 次以上 → **tested**（已测试）
- 实战 5 次以上 + 胜率 ≥ 60% + 平均分 ≥ 70 → **mature**（成熟）

## 第五章：命令一卡通（大白话版）

| 命令 | 一句话说它在干嘛 | 怎么用 |
|------|----------------|--------|
| `init` | 准备开工，创建需要的文件夹 | `skill-forge init` |
| `ingest` | 把你的案例/材料存进来 | `skill-forge ingest --type case --title "标题" --file 文件路径` |
| `distill` | 把材料炼成 Skill（核心功能） | `skill-forge distill --material 材料ID --problem "要解决的问题"` |
| `inspect` | 检查一个外部文件，看能不能吸收 | `skill-forge inspect --file 文件路径` |
| `melt` | 把外部文件熔炼成 Skill | `skill-forge melt --file 文件路径` |
| `drill` | 模拟客户跟你对练 | `skill-forge drill --skill SkillID --persona "客户类型"` |
| `review` | 复盘一次真实聊天 | `skill-forge review --file 聊天记录文件 --result "结果"` |
| `field-log` | 记一笔实战成绩 | `skill-forge field-log --skill SkillID --result "成交"` |
| `recommend` | 这个客户来了，推荐用哪个 Skill | `skill-forge recommend --file 当前聊天记录` |
| `search` | 搜一下自己有那些 Skill | `skill-forge search --query "价格"` |
| `propose-update` | 让 AI 建议怎么改进 Skill | `skill-forge propose-update --skill SkillID` |
| `diff` | 对比 Skill 的两个版本有啥不同 | `skill-forge diff --skill SkillID --from 1.0.0 --to 1.1.0` |
| `history` | 看一个 Skill 的修改历史 | `skill-forge history --skill SkillID` |
| `rollback` | 回退到之前的版本 | `skill-forge rollback --skill SkillID --to 1.0.0` |
| `promote` | 手动给 Skill 升一级 | `skill-forge promote --skill SkillID --to mature` |
| `validate` | 检查一个文件格式对不对 | `skill-forge validate --file 文件路径` |

## 第六章：常见问题

### Q：我不会用命令行，怎么办？

命令行没有想象中可怕。你就当成这样：
- **Win+R → 输入 cmd → 回车** = 打开一个"黑框框"
- 在黑框框里输入命令 → 回车 = 让它干活

你只需要会三件事：
1. 打开黑框框
2. 复制粘贴命令
3. 看它输出的结果

### Q：Workbuddy 生成的 Skill 怎么存回来？

Workbuddy 的回答里有一段以 `---` 开头和结尾的内容，那就是 YAML Front Matter。全选复制，在 `data/skills/draft/` 目录下新建一个 `.md` 文件（名字随便起，比如 `my-skill.md`），粘贴进去保存即可。

### Q：为什么要先导入再提炼？不能直接提炼吗？

可以直接提炼——用 `melt` 命令就行。`melt` 直接把文件转成 Skill，不需要先导入。

导入的目的是"把原始材料存一份"——以后你想看当时基于什么提炼的，能找到来源。

### Q：Skill 的状态会自动升吗？

会的。每次演练和复盘后，系统自动检查是否满足晋升条件：

| 状态 | 怎么达到 | 通俗理解 |
|------|---------|---------|
| draft（草稿） | 刚提炼出来 | 刚写了个草稿 |
| trained（已训练） | 演练 ≥ 3 次 + 平均分 ≥ 60 | 练了几遍，基本熟了 |
| tested（已测试） | 实战 ≥ 1 次 | 真刀真枪用过至少一次 |
| mature（成熟） | 实战 ≥ 5 次 + 胜率 ≥ 60% | 多次验证，稳定有效 |
| retired（退役） | 手动标记 | 这个 Skill 过时了 |

### Q：Skill 会越来越好用吗？

会的。每次复盘后你会得到改进建议，你可以手动更新 Skill 的内容（修改 .md 文件）。每次修改版本号 + 0.1，系统会保留历史版本，你可以随时回退。

## 第七章：这东西是怎么造出来的？（给好奇的你）

如果你只是用工具，看完上面就够了。如果你好奇"这东西怎么造出来的"，往下看。

### 7.1 从零到一

Skill Forge 一开始只是一个想法：**销售团队有大量经验（聊天记录、案例、培训材料），但这些经验散落在各处，没法复用。**

最开始，这就是一个空目录。经过反复迭代，变成了现在的 35 个文件、近 3000 行代码。

### 7.2 用了什么技术？

| 技术 | 干嘛用的 | 大白话 |
|------|---------|--------|
| Python 3.10+ | 写代码的语言 | 就是写这个工具用的语言 |
| typer | 处理命令行的框架 | 让你能敲 `skill-forge xxx` 这种命令 |
| PyYAML | 处理 YAML 格式 | 读写 Skill 文件里的 `---` 之间的内容 |
| Pydantic | 校验数据格式 | 确保你写的 Skill 格式是对的，没有漏填必填项 |
| rich | 美化命令行输出 | 让黑框框里的字不是纯白的，有颜色、有表格 |
| python-dotenv | 读取配置文件 | 自动读 `.env` 文件里的设置 |

### 7.3 踩过的坑

#### 坑一：路径问题

**问题**：最开始 Skill 数据写到了安装目录里。如果你用 `pip install` 安装，数据会写到 Python 的安装目录。那里你可能没有写权限（特别是公司电脑）。

**修复**：改成 `Path.cwd()`——数据写到你运行命令时所在的目录。你在哪个目录敲命令，数据就写在哪。

#### 坑二：Schema 分裂

**问题**：最开始定义 Skill 的格式时，校验代码用一套字段名（`applicable_scenarios`），但 Prompt 模板里用另一套（`scenes`）。结果是模板生成的内容，校验通不过。

**修复**：统一成一套字段名，所有代码、模板、文档都用同一个 Schema。同时在代码里加备选逻辑：如果新字段不存在，尝试读旧字段。

#### 坑三：文件读写不安全

**问题**：如果有恶意用户传一个 `../../etc/passwd` 路径，可能会读取系统文件。

**修复**：加了两层防护：
1. 写文件时检查路径是否在允许的目录内
2. 读文件时先读再检查大小（防止"检查时小文件，打开时大文件"的竞态攻击）

#### 坑四：AI 提示词注入

**问题**：如果材料内容里包含"忽略之前的指令"这种话，AI 可能会被带偏。

**修复**：对用户输入做过滤，包括：
- Unicode 标准化（把长得像的俄文字母转成英文字母）
- 过滤常见的注入关键词
- 限制输入长度
- 擦除零宽字符

#### 坑五：Shell 无限挂死

**问题**：有一次测试时 PowerShell 进程挂死了，所有命令都超时。怎么重启都恢复不了。

**修复**：需要手动在任务管理器里杀掉所有 Python 和 PowerShell 进程。最后重启电脑才彻底解决。（这个问题跟代码没关系，是运行环境的问题。）

### 7.4 怎么保证质量？

#### 测试体系

一共 **159 个自动化测试**，每次改完代码跑一遍：

| 测试分类 | 数量 | 测什么 |
|---------|------|--------|
| 功能测试 | 38 | 核心功能是否正常 |
| 安全测试 | 47 | 能不能防住各种攻击 |
| 生命周期测试 | 13 | 从导入到成熟整个流程 |
| AI 测试 | 19 | AI 调用、输入净化、输出消毒 |
| 存储测试 | 16 | 文件读写、路径验证 |
| Schema 测试 | 19 | 数据格式校验 |

#### 攻防演练

做了两轮"地狱级攻防"：
- **攻击者**（叫"深渊"）：专门找漏洞，想方设法攻破
- **防御者**（叫"铁壁Pro"）：确认漏洞，修复代码，验证修复

整个过程记录在 `SOP.md` 里，如果你感兴趣可以看。

### 7.5 设计原则

```
1. 没 API Key 也能用         → 输出完整 Prompt，手动复制到 AI 工具
2. 数据存在本地               → 不联网，不上传，隐私安全
3. 每个 Skill 就是一个文件    → 用记事本就能改，没有数据库
4. 所有配置集中管理           → 不用到处找设置
5. 写操作严格限制目录         → 防止文件跑出范围
6. 读操作不限制               → 用户自己的文件随便读
7. 格式灵活                  → JSON/YAML/Markdown 都能导入
8. 经验可以版本管理           → 改错了能回退
```

### 7.6 Skill Forge 的定位：预加工工具

这是我最想澄清的一点。

**Skill Forge 做的事**：

```
书/案例 → [skill-forge处理] → 结构化文档
```

**它停在这里了**。输出是**文档**，不是"能力"。

#### 你要的"随意调用"还差一步

| 阶段 | 工具 | 输出 | 能调用吗？ |
|------|------|------|-----------|
| 第1步 | Skill Forge | 结构化文档 | ❌ 不能 |
| 第2步 | ？（缺） | AI可调用格式 | ✅ 能 |

#### 第2步有3种做法

**做法1：转成 WorkBuddy Skill（最简单）**
```
Skill Forge 输出 → 手动/自动转成 Skill → 你能直接调用
```
- 优点：快，能用
- 缺点：只限 WorkBuddy

**做法2：接入 RAG 知识库（最灵活）**
```
书 → 切片 → 向量数据库 → AI 检索后回答
```
- 优点：任何 AI 都能用，支持模糊查询
- 缺点：需要搭建向量库

**做法3：转成 MCP 工具（最通用）**
```
书 → 封装成 MCP Server → 任何 AI 都能调
```
- 优点：标准化，通用
- 缺点：开发成本高

#### 怎么选？

| 你的资料类型 | 推荐路径 |
|-------------|---------|
| 案例/经验类（销售话术、客户异议） | Skill Forge → 转成 Skill → 调用 |
| 理论/知识类（产品手册、行业报告） | 直接用 RAG |
| 混合类型 | Skill Forge 处理案例部分 + RAG 处理理论部分 |

## 写在最后

Skill Forge 不是一个完美的工具，但它做的事情很简单：**把零散的经验变成结构化的、可复用的话术**。

对于销售团队来说，这意味着：
- 销冠的经验不会随着他离职而流失
- 新手不用从零摸索，照着 Skill 练就行
- 每次实战都在积累，而不是重复犯同样的错误
