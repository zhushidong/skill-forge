# 评分标准

## Drill 评分（演练评分）

### 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **diagnosis** | 25% | 是否准确识别客户问题 |
| **response_quality** | 25% | 回应质量和话术水平 |
| **next_step_control** | 25% | 是否推进到下一步 |
| **risk_control** | 25% | 是否避免禁用行为 |

### 评分等级

每个维度 0-100 分：

| 分数范围 | 等级 | 说明 |
|---------|------|------|
| 90-100 | 优秀 | 完美执行，可作为标杆 |
| 70-89 | 良好 | 基本执行到位，有小瑕疵 |
| 50-69 | 及格 | 执行有偏差，需要改进 |
| 30-49 | 不及格 | 执行有明显问题 |
| 0-29 | 失败 | 完全没有执行或执行错误 |

### 细项评分

#### Diagnosis（诊断能力）

| 分数 | 标准 |
|------|------|
| 90-100 | 准确识别客户核心问题，区分表面需求和真实需求 |
| 70-89 | 识别了主要问题，但遗漏次要问题 |
| 50-69 | 识别了问题但不够准确 |
| 30-49 | 误判问题 |
| 0-29 | 完全没有诊断 |

#### Response Quality（回应质量）

| 分数 | 标准 |
|------|------|
| 90-100 | 话术自然、有说服力、符合品牌调性 |
| 70-89 | 话术基本有效，但不够流畅 |
| 50-69 | 话术生硬或有明显问题 |
| 30-49 | 话术无效或适得其反 |
| 0-29 | 完全没有回应或回应错误 |

#### Next Step Control（推进控制）

| 分数 | 标准 |
|------|------|
| 90-100 | 成功推进到明确的下一步 |
| 70-89 | 推进了但下一步不够明确 |
| 50-69 | 试图推进但失败 |
| 30-49 | 对话停滞 |
| 0-29 | 对话倒退或客户流失 |

#### Risk Control（风险控制）

| 分数 | 标准 |
|------|------|
| 90-100 | 完全避免所有禁用行为 |
| 70-89 | 避免了主要禁用行为 |
| 50-69 | 触碰了非关键禁用行为 |
| 30-49 | 触碰了关键禁用行为 |
| 0-29 | 严重违反禁用行为 |

### Drill 结果分类

| 结果 | 条件 | 说明 |
|------|------|------|
| **成功** | avg_score >= 70 且 next_step_control >= 70 | 成功推进 |
| **部分成功** | avg_score >= 50 且 next_step_control >= 50 | 有进展但未完全成功 |
| **失败** | avg_score < 50 或 next_step_control < 50 | 未能推进 |
| **严重失败** | avg_score < 30 或 risk_control < 30 | 触碰禁用行为或严重失误 |

## Review 评分（复盘评分）

### 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **adherence** | 30% | 是否遵循 Skill 步骤 |
| **outcome** | 30% | 最终结果 |
| **improvement** | 20% | 相比上次是否有进步 |
| **skill_defect** | 20% | Skill 本身的缺陷（越低越好） |

### 评分等级

| 分数范围 | 等级 | 说明 |
|---------|------|------|
| 90-100 | 优秀 | 完美执行并产生正面结果 |
| 70-89 | 良好 | 基本执行到位，结果可接受 |
| 50-69 | 及格 | 执行有偏差，结果一般 |
| 30-49 | 不及格 | 执行有明显问题，结果负面 |
| 0-29 | 失败 | 完全没有执行或结果严重负面 |

### 细项评分

#### Adherence（执行遵循度）

| 分数 | 标准 |
|------|------|
| 90-100 | 完全遵循所有步骤 |
| 70-89 | 遵循主要步骤，遗漏次要步骤 |
| 50-69 | 遗漏重要步骤 |
| 30-49 | 大部分步骤未执行 |
| 0-29 | 完全没有执行 |

#### Outcome（结果）

| 分数 | 标准 |
|------|------|
| 90-100 | 成交或明确推进 |
| 70-89 | 推进到下一步 |
| 50-69 | 对话继续但无明确进展 |
| 30-49 | 对话停滞或客户犹豫 |
| 0-29 | 客户流失或明确拒绝 |

#### Improvement（改进）

| 分数 | 标准 |
|------|------|
| 90-100 | 显著优于上次 |
| 70-89 | 略优于上次 |
| 50-69 | 与上次持平 |
| 30-49 | 不如上次 |
| 0-29 | 显著退步 |

#### Skill Defect（Skill 缺陷）

| 分数 | 标准 |
|------|------|
| 90-100 | Skill 无缺陷 |
| 70-89 | 有小瑕疵但不影响使用 |
| 50-69 | 有明显缺陷需要改进 |
| 30-49 | 有严重缺陷影响效果 |
| 0-29 | Skill 完全不适用 |

### Review 结果分类

| 结果 | 条件 | 说明 |
|------|------|------|
| **成交** | outcome >= 90 | 客户成交 |
| **推进** | outcome >= 70 | 成功推进到下一步 |
| **持平** | outcome >= 50 | 对话继续但无进展 |
| **流失** | outcome < 50 | 客户流失 |
| **严重失败** | outcome < 30 且 risk_control < 30 | 严重失误导致流失 |

## Recommend 评分（推荐评分）

### 评分维度

| 维度 | 说明 | 权重 |
|------|------|------|
| **relevance** | 推荐 Skill 与当前场景的相关度 | 40% |
| **confidence** | 推荐置信度（0-1） | 30% |
| **risk_warning** | 是否包含风险提醒 | 15% |
| **explanation** | 推荐理由是否清晰 | 15% |

### 评分等级

| 分数范围 | 等级 | 说明 |
|---------|------|------|
| 90-100 | 精准推荐 | 完美匹配，高置信度 |
| 70-89 | 合理推荐 | 基本匹配，有明确理由 |
| 50-69 | 一般推荐 | 部分匹配，理由不够充分 |
| 30-49 | 错误推荐 | 匹配度低或理由不成立 |
| 0-29 | 严重错误 | 完全不匹配或有害推荐 |

### 推荐解释模板

```markdown
## 推荐解释

### 推荐 Skill
[Skill 名称] (ID: [skill-id])

### 推荐原因
1. 当前对话命中客户信号：[信号列表]
2. 客户仍有购买意向：[证据]
3. 当前阶段适合此策略：[原因]
4. 历史相似场景使用成功率：[百分比]

### 风险提醒
- [风险 1]
- [风险 2]

### 不推荐其他 Skill 的原因
- [Skill A]：[不匹配原因]
- [Skill B]：[不匹配原因]

### 置信度
[0-100]%
```

## 综合评分公式

### Skill 健康度评分

```
health_score = (
    drill_avg_score * 0.3 +
    review_avg_score * 0.3 +
    win_rate * 0.2 +
    freshness_score * 0.2
)
```

其中：

- `drill_avg_score`: 近 5 次 drill 平均分
- `review_avg_score`: 近 5 次 review 平均分
- `win_rate`: wins / (wins + losses)
- `freshness_score`: 基于最近使用时间的衰减分数

### 健康度等级

| 分数范围 | 等级 | 建议操作 |
|---------|------|---------|
| 80-100 | 健康 | 继续使用 |
| 60-79 | 一般 | 考虑优化 |
| 40-60 | 警告 | 需要改进 |
| 0-40 | 危险 | 考虑退役 |
